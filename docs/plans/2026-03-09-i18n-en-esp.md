# i18n EN/ESP Toggle — Implementation Plan

> **For Claude:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Add a client-side language toggle (ES/EN) to the HYDRA cloud dashboard so all UI text switches instantly between Spanish and English.

**Architecture:** A `static/js/i18n.js` file contains a `TRANSLATIONS` dictionary and a `t(key)` helper. HTML elements get `data-i18n` attributes. `setLang()` walks the DOM and updates text. Dynamic JS strings in `dashboard.js` use `t()`. Toggle button in header next to dark mode. Language persisted in `localStorage`.

**Tech Stack:** Vanilla JS (no dependencies), HTML data attributes, localStorage

**Security note:** All translation content is developer-controlled (hardcoded in i18n.js), never user-supplied. The `data-i18n-html` attribute is used only for our own static content containing HTML formatting (bold, links to FRED). No user input is ever rendered as HTML.

### Task 1: Create `i18n.js` — Core Translation Infrastructure

**Files:**
- Create: `static/js/i18n.js`

**Step 1: Create the i18n module**

Create `static/js/i18n.js` with:
- `currentLang` variable (from localStorage, default 'es')
- `TRANSLATIONS` object with `es` and `en` keys containing all UI string translations
- `t(key)` function that returns the translation for the active language
- `setLang(lang)` function that updates all `[data-i18n]` elements via `.textContent` and all `[data-i18n-html]` elements via `.innerHTML` (safe because content is developer-controlled, never user-supplied)
- `toggleLang()` function
- `updateLangToggle()` function to style the toggle button
- DOMContentLoaded listener to apply language on load

The translations dictionary should cover all sections:

**Header:** market status labels, offline banner
**Tabs:** Dashboard, News, Roadmap, Algorithm, ML
**Hero:** stat labels, contexts, features, disclaimer
**Perf Banner:** today/overall labels, beating/behind text
**Metric Cards:** all labels
**Regime/Overlay:** labels, explanation content
**Positions:** strategy labels, loading/empty states
**P2P/Annual Returns/Trade Analytics:** titles, stat labels
**Social Feed:** all labels, filters, empty state
**Roadmap:** headings, phases, statuses, decision titles
**Algoritmo page:** section titles, pipeline labels
**ML page:** KPI labels, pipeline, phase titles
**Footer:** disclaimer

For long-form Algoritmo page content (sections 01-09), Roadmap descriptions, and ML descriptions: these contain substantial HTML with links, tables, formatting. Use `data-i18n-html` for these sections. The HTML content is all developer-controlled static content.

**Step 2: Commit**

```bash
git add static/js/i18n.js
git commit -m "feat: add i18n translation infrastructure with ES/EN dictionary"
```
